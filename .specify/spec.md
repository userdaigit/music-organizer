# Music Organizer v2.0 - 规格说明 (Spec)

> 状态：草案 (Draft)
> 创建：2026-07-06
> 此文件描述 v2.0 要构建**什么**以及**为什么**，不涉及技术实现。

## 背景

v1.x 通过迭代式开发完成，积累了以下技术债：
- 版本号曾散落 5+ 文件（v1.1.0 已修复）
- 无正式测试套件（v1.1.0 已修复）
- 无 CI/CD（v1.1.0 已修复）
- Windows 启动脚本缺失（v1.1.0 已修复）

v2.0 的目标是将工具从"能用"提升到"好用"。

## v2.0 功能规格

### MUST HAVE（必须实现）

1. **配置文件支持**
   - 支持 YAML 配置文件，集中管理所有参数
   - 命令行参数覆盖配置文件
   - 配置文件示例：`config.example.yaml`

2. **增量整理**
   - 记录已整理文件清单（manifest）
   - 二次运行时跳过已整理的文件
   - 支持 `--force` 强制重新整理

3. **日志系统**
   - 替换 print 为 logging 模块
   - 支持 `--verbose` / `--quiet` 控制输出级别
   - 日志文件自动写入输出目录

### SHOULD HAVE（应该实现）

4. **GUI 配置工具**
   - 简单的 Web UI（Flask），可视化配置参数
   - 不是必须，但降低使用门槛

5. **更多音频格式**
   - 支持 DSD (.dsf/.dff)
   - 支持 MusePack (.mpc)

### NICE TO HAVE（可以实现）

6. **多语言界面**
   - 中英文 CLI 输出

7. **插件系统**
   - 支持自定义命名规则插件

## 非目标（v2.0 不做）

- 不做音乐播放器
- 不做歌词下载
- 不做专辑封面下载（可作为独立工具）

## 验收标准

- [ ] pytest 测试覆盖率 >= 80%
- [ ] CI 在 Python 3.9/3.11/3.13 全部通过
- [ ] README 文档完整更新
