# Music Organizer - 规格说明 (Spec)

> 状态：v1.2.0 需求已追加
> 创建：2026-07-06
> 更新：2026-07-06（v1.2.0 多刮削源需求）
> 此文件描述要构建**什么**以及**为什么**，不涉及技术实现。

## 背景

v1.x 通过迭代式开发完成。v1.1.0 已完成 Converge 整改（版本集中管理、测试套件、CI/CD、进度条、Windows 脚本）。
v1.2.0 聚焦：增强华语音乐刮削能力，解决 MusicBrainz 对华语音乐覆盖有限的问题。

## v1.2.0 功能规格（本次开发）

### MUST HAVE（必须实现）

1. **AcoustID API KEY 有效性检查**
   - 启动时检测 API KEY 是否为默认测试 KEY（`lmv7m8k7Fe`）
   - 若为默认 KEY 或无效 KEY，提示"未配置 API KEY，或 API KEY 无效，无法使用音频指纹识别"
   - 用户配置有效 KEY 后，自动启用指纹识别功能
   - KEY 有效性通过向 AcoustID 发送测试请求验证

2. **多刮削源支持**

   **技术调研结论（2026-07-06）：**

   | 刮削源 | 可用性 | 法律状态 | 是否纳入 |
   |---|---|---|---|
   | MusicBrainz | 官方公开 API，已有 | CC0 许可，合规 | 已有，保留 |
   | AcoustID | 官方公开 API，已有 | 需 API KEY，合规 | 已有，增强 KEY 检查 |
   | 酷狗音乐 | 移动端搜索接口可用，无需认证 | 非官方接口，可能随时失效 | 纳入，标注风险 |
   | 网易云音乐 | 主要非官方项目（Binaryify/NeteaseCloudMusicApi）已因版权删库停更 | 非官方，法律风险 | 不纳入 |
   | QQ音乐 | 仅有逆向工程项目，无可靠公开 API | 非官方，法律风险 | 不纳入 |

   **功能要求：**
   - 新增酷狗音乐搜索刮削源（搜索歌曲名获取歌手、专辑、年份等元数据）
   - 刮削顺序：MusicBrainz → 酷狗音乐 → AcoustID 指纹
   - 任一源获取到有效信息即停止，不重复刮削
   - 刮削失败时优雅降级，不影响整理流程
   - 非官方接口标注"可能失效"提示

### SHOULD HAVE（应该实现）

3. **刮削源可配置**
   - 通过命令行参数 `--scrapers` 指定刮削源顺序
   - 默认顺序：musicbrainz,kugou
   - 不需要的刮削源可跳过

### NICE TO HAVE（可以实现）

4. **网易云/QQ音乐刮削**
   - 待官方公开 API 可用后再纳入
   - 当前因非官方项目已删库或不可靠，不实现

## v2.0 功能规格（后续开发，本次不实现）

1. **配置文件支持** - YAML 配置文件
2. **增量整理** - manifest 清单
3. **日志系统** - logging 模块
4. **GUI 配置工具** - Flask Web UI
5. **更多音频格式** - DSD、MusePack

## 非目标

- 不做音乐播放器
- 不做歌词下载
- 不做专辑封面下载
- 不做音乐下载（仅刮削元数据）
- 不纳入已删库的非官方 API（网易云、QQ音乐逆向项目）
