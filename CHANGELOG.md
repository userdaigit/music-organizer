# 变更日志

## v1.2.0 (2026-07-06)

### 新功能
- **酷狗音乐刮削源**：新增酷狗音乐搜索接口作为华语音乐刮削源，弥补 MusicBrainz 对华语音乐覆盖不足
- **多刮削源链式调用**：MusicBrainz → 酷狗音乐 → AcoustID 指纹，按优先级依次尝试，任一源获取到有效信息即停止
- **AcoustID API KEY 有效性检查**：启动时检测 KEY 状态，默认测试 KEY 或无效 KEY 时提示"未配置API KEY，或API KEY无效，无法使用音频指纹识别"
- **独立进度条**：每个刮削源（MusicBrainz、酷狗音乐）显示独立进度条

### 技术调研结论
- 网易云音乐：主要非官方 API 项目（Binaryify/NeteaseCloudMusicApi）已因版权删库停更，不纳入
- QQ音乐：仅有逆向工程项目，无稳定公开 API，不纳入
- 酷狗音乐：移动端搜索接口可用，无需认证，纳入但标注"非官方接口，可能随时失效"

### 改进
- 刮削失败时优雅降级，不影响整理流程
- 指纹识别步骤优化：无需指纹识别时跳过
- SDD 流程：spec/plan/tasks 文档更新

## v1.1.0 (2026-07-06)

### 新功能
- **进度条显示**：所有耗时步骤（扫描、元数据提取、歌手规范化、网络刮削、指纹识别、复制文件）均显示 pip 风格百分比进度条
- **输出目录自动创建**：输出目录不存在时自动创建，权限不足时提示手动创建
- **源目录检查**：源目录不存在时报错退出，避免无效运行
- **版本号集中管理**：新增 `version.py`，所有文件统一引用，避免散落多处
- **Windows 启动脚本**：新增 `organize.ps1`，支持 PowerShell 环境一键启动

### 工程改进
- **正式测试套件**：新增 `tests/` 目录，106 个 pytest 测试覆盖核心逻辑（feat.解析、编码修复、进度条、路径构建等）
- **CI/CD**：新增 GitHub Actions 配置，Python 3.9/3.11/3.13 矩阵自动测试
- **分发自动化**：新增 `sync_dirs.ps1`，自动同步 github-release 到 script/docker 目录
- **歌手规范化 API**：新增 `normalize_one()` 方法，支持逐个显示进度

### 文档更新
- 命令行参数和帮助信息增加路径替换提示
- README 中 User-Agent 修改位置更新为引用 version.py
